# Versioning
**Current Version:** v0.1.1 (Prototype)

## Versioning Scheme
- `v0.x.y` = Prototype phase
  - `x` increases for major prototype milestones (new systems)
  - `y` increases for fixes/polish within the milestone

## Release Tags (recommended)
- Tag Git commits as `v0.1.1`, `v0.1.2`, etc.
- Keep patch notes in `docs/PATCH_NOTES.md` (or per-release files).
